# X 浏览历史

一个本地优先的 Chrome / Edge 扩展：自动记录你在 X 上真正看过的帖子，并提供可搜索的浏览历史。

![X 浏览历史图标](icons/icon-128.png)

## 能做什么

- 在帖子进入屏幕并停留一段时间后自动记录，快速划过的不算。
- 在扩展弹窗里快速搜索最近看过的帖子。
- 在完整历史页按关键词、作者、链接和时间范围筛选。
- 支持暂停记录、设置停留时间、设置历史上限、导入和导出备份。
- 历史记录只保存在浏览器本机，不上传到服务器。

## 安装

1. 下载 [最新版 ZIP](https://github.com/SilenceGYL/X-Browsing-History-Review/archive/refs/heads/main.zip) 并解压。
2. 打开 Chrome 的 `chrome://extensions/`，或 Edge 的 `edge://extensions/`。
3. 打开右上角的“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择解压后的插件目录。
6. 打开或刷新 `https://x.com/`，正常浏览即可开始记录。

## 使用说明

- 默认情况下，帖子在屏幕中显示超过 `1.2` 秒才会被记录。
- 扩展图标右上角会显示累计记录数，可用来确认记录是否生效。
- 从其他标签页切回 X，或在 X 内部切换页面后，记录会自动恢复。
- 点击浏览器工具栏里的扩展图标，可以快速查询最近浏览。
- 点击“打开完整历史”，可以筛选、导入、导出或清空记录。
- 卸载扩展会删除浏览器本机里的记录。卸载前请先导出备份。

## 实现方式

- `src/content.js` 观察 X 页面里的帖子卡片，按可见比例和停留时间判断是否真正浏览。
- `src/background.js` 使用浏览器内置的 IndexedDB 保存记录，避免把历史发送到外部服务。
- `popup.html` 提供快速查询入口，`history.html` 提供完整历史管理页。

## 当前限制

- X 修改页面结构后，帖子识别规则可能需要跟着更新。
- 当前只记录帖子文本、作者、链接、发布时间、浏览时间和媒体类型，不会离线保存图片或视频。

## 没有记录时

1. 在扩展管理页找到“X 浏览历史”，点击一次刷新按钮。
2. 回到已经打开的 X 页面，再刷新一次网页。浏览器不会把刚加载或刚更新的扩展自动注入旧页面。
3. 在一条帖子上停留约 `1.2` 秒，观察扩展图标右上角是否出现数字。

更新扩展后，旧的 X 标签页可能短暂显示一次“扩展上下文已失效”错误。`0.1.4` 起旧页面会自动停止运行；刷新 X 页面后即可继续记录。
